# 예산관리 엑셀 자동생성 스크립트 설계

작성일: 2026-07-18

## 목적

예산 항목 목록을 받아 예산관리용 xlsx 파일을 생성한다. 생성된 파일은
"만들고 끝나는 보고서"가 아니라 **엑셀에서 계속 수정하며 쓰는 관리 파일**이다.
따라서 잔액·집행률은 엑셀 수식으로, 경고 색상은 조건부 서식으로 넣어
사용자가 집행액만 고쳐도 나머지가 자동으로 따라오게 한다.

```
  test_budget_excel.py  검증 테스트

```

## 공개 인터페이스

```python
create_budget_file(items, output_path) -> Path
```

- `items`: `(항목명: str, 예산액: int|float, 집행액: int|float)` 시퀀스의 리스트
- `output_path`: 저장 경로 (str 또는 Path)
- 반환: 저장된 파일의 `Path`

모듈을 직접 실행하면 (`python budget_excel.py`) 샘플 데이터로
`sample_budget.xlsx`를 생성한다.

## 시트 구조

| 열 | 내용 | 값의 종류 | 표시 형식 |
|----|------|-----------|-----------|
| A | 항목명 | 문자열 | 기본 |
| B | 예산액 | 숫자 | `#,##0` |
| C | 집행액 | 숫자 | `#,##0` |
| D | 잔액 | 수식 `=B{r}-C{r}` | `#,##0` |
| E | 집행률 | 수식 `=IF(B{r}=0,0,C{r}/B{r})` | `0.0%` |

- 1행은 헤더: 굵게, 회색 배경, 가운데 정렬
- 데이터는 2행부터 시작
- 마지막 데이터 행 다음에 합계 행:
  - A열 `"합계"`, B·C열 `SUBTOTAL(109, ...)`, D·E열은 데이터 행과 동일한 수식
  - `SUBTOTAL`을 쓰는 이유: 사용자가 자동 필터로 일부 항목을 숨겨도
    보이는 항목만 합산되어 합계가 의미를 유지한다
- 틀 고정: `A2` (헤더 고정)
- 열 너비: 항목명 넓게, 금액 열은 자릿수에 맞게 고정 폭

## 조건부 서식

데이터 행 범위 `A2:E{마지막데이터행}`에 수식 기준 규칙 2개를 등록한다.
합계 행은 범위에서 제외한다 — 전체 집행률이 80%를 넘었다고 합계 줄까지
빨갛게 칠하면 어떤 개별 항목이 문제인지 읽기 어려워지기 때문이다.

| 우선순위 | 수식 | 채우기 |
|----------|------|--------|
| 1 | `=$E2>=1` | 빨강 (`FFC7CE`) |
| 2 | `=$E2>=0.8` | 노랑 (`FFEB9C`) |

- 절대/상대 참조 주의: `$E2` — 열은 고정, 행은 상대.
  이래야 한 규칙이 각 행에 자기 행 기준으로 적용된다.
- 빨강 규칙을 먼저 등록한다. openpyxl은 등록 순서대로 우선순위를 매기고,
  100% 이상인 행은 두 조건을 모두 만족하므로 순서가 결과를 결정한다.
- 색칠 범위는 A~E 행 전체. 집행률 셀 하나만 칠하는 것보다
  위험 항목을 한눈에 스캔하기 쉽다.
- 경계 처리: `>=` — 정확히 80.0%인 항목도 경고에 포함된다.

## 샘플 데이터

경계값을 모두 포함해 파일을 열었을 때 규칙이 눈으로 검증되도록 구성한다.

| 항목명 | 예산액 | 집행액 | 집행률 | 기대 색 |
|--------|--------|--------|--------|---------|
| 사무용품비 | 1,000,000 | 450,000 | 45.0% | 없음 |
| 교육훈련비 | 2,000,000 | 1,600,000 | 80.0% | 노랑 (하단 경계) |
| 출장여비 | 3,000,000 | 2,760,000 | 92.0% | 노랑 |
| 회의비 | 500,000 | 500,000 | 100.0% | 빨강 (경계) |
| 소모품비 | 800,000 | 920,000 | 115.0% | 빨강 (초과) |
| 예비비 | 0 | 0 | 0.0% | 없음 (0으로 나누기 방어) |

## 에러 처리

- 항목명이 빈 문자열이거나 공백뿐이면 `ValueError`
- 예산액 또는 집행액이 숫자가 아니면 `ValueError`
- 예산액 또는 집행액이 음수면 `ValueError`
- `items`가 비어 있으면 `ValueError` — 데이터 행이 없으면 조건부 서식
  범위와 `SUBTOTAL` 범위를 만들 수 없고, 빈 파일은 사용자 의도가 아닐
  가능성이 높다
- 예산액 0은 허용한다. 미배정 항목은 실제로 존재하며, 집행률 수식이
  `IF`로 이미 방어하고 있다

## 검증 방법

`test_budget_excel.py`에서 파일을 생성한 뒤 openpyxl로 다시 읽어 확인한다.

- 헤더 문자열과 데이터 행 값
- D·E열에 계산된 숫자가 아니라 **수식 문자열**이 들어갔는지
- 조건부 서식 규칙 2개가 올바른 범위·수식·색·순서로 등록됐는지
- 숫자 표시 형식과 틀 고정
- 잘못된 입력에 대한 `ValueError`

**검증의 한계:** 조건부 서식의 실제 렌더링 색과 수식의 계산 결과값은
엑셀이 파일을 열 때 계산한다. openpyxl은 규칙과 수식이 파일에 올바르게
*기록됐는지*까지만 확인할 수 있다. 노란색이 실제로 화면에 보이는지는
사람이 파일을 열어 확인해야 한다.
